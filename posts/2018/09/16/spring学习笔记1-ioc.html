# 什么是IoC

IoC: Inversion of Control, 控制反转，指应用程序将对象的控制权交出去，由容器框架来管理对象的生命周期。

在传统的应用程序中，需要开发人员手动进行对象管理。比如在需要某个类时new一个对象实例，不用时进行销毁。而在Spring框架中，对象的生命周期由Spring管理。如图所示，当应用程序需要某个对象时，直接从Spring容器中获得相应的对象（Spring Beans）即可。
Spring Beans 通过依赖注入方式实现，从某种意义上来说，依赖注入(DI - Dependency Injection)和IoC是等价的，细究起来，IoC是Spring的特点，而DI是IoC的实现方式。

IoC的出现降低了依赖间的耦合度，举一个知乎的例子:

假如有一个 船(C)类 ，一个 桨(J) 类

class C{
  J j = new J() ;
}

如果船要干什么事，肯定需要桨的参与。所以是十分“依赖”桨；
这时出了新的需求需要重构：这时候我们需要控制桨的长度为10在构造方法中。我们需要这么写：

class C{
  J j = new J(10) ;
}

如果有一个改动同时需要修改桨构造方法，又需要修改船其中的new J()方法(修改两处)。这时候就设计者就思考，为什么我们加入一个特性需要更改两个类中代码（这也就是耦合度高）！所以我们要解耦要依赖注入；

# 1. 使用XML

在 src目录下创建spring-injection.xml文件，注入bean：

<bean id="kkyDao" class="com.kkyan.IoC.kkyDaoImpl"> </bean>
<bean id="kkyService" class="com.kkyan.IoC.kkyServiceImpl">
	<property name="kkyDao" ref="kkyDao"></property>
</bean>
